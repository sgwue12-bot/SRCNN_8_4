SRCNN_8_4 RTL Implementation Project
====================================

Overview
--------
This project implements an SRCNN_8_4 image super-resolution accelerator in Verilog RTL.
The design targets the DSD26 term project environment and is structured for KV260/FPGA
implementation and simulation verification.

The implemented SRCNN network has three 3x3 convolution layers:

- Layer1: 1 input channel  -> 8 output channels
- Layer2: 8 input channels -> 4 output channels
- Layer3: 4 input channels -> 1 output channel

The total parallel MAC/DSP count is:

- Layer1: 72 MACs
- Layer2: 288 MACs
- Layer3: 36 MACs
- Total: 396 MACs / DSPs


Main Design Idea
----------------
The main goal of this implementation is to process three 150x150 input images with a
streaming datapath instead of storing every intermediate feature map as a full frame.

Layer1 output is directly fed into the Layer2 line buffers, and Layer2 output is directly
fed into the Layer3 line buffers. Once the required rows are available, the next layer can
start processing without waiting for an entire feature map to be written and read back.


Key RTL Components
------------------
The uploaded zip file contains the final RTL source files, including:

- top_srcnn.v
  Top-level wrapper that connects SRCNN output to the output memory packing logic.

- SRCNN.v
  Main datapath integration module. It connects input memory, weight/bias memory,
  global controller, line buffers, layer controllers, and convolution layers.

- input_mem_3img.v
  BRAM-style input memory for three Y-channel 150x150 images.

- weight_bias_mem_8_4.v
  Weight and bias ROM for the SRCNN_8_4 model.

- line_buffer_3x150_1ch.v
  Streaming 3x3 window generator using two 150-cycle delay lines and 3-column shift
  registers.

- layer1/layer1_conv_8ch.v
  Layer1 convolution block that generates 8 output channels from a 1-channel 3x3 window.

- layer2/layer2_conv_4ch.v
  Layer2 convolution block that generates 4 output channels from 8-channel 3x3 windows.

- layer3/layer3_conv_1ch.v
  Layer3 convolution block that generates the final 1-channel output.

- layer2/layer2_linebuf_8ch.v and layer3/layer3_linebuf_4ch.v
  Multi-channel line buffer modules for Layer2 and Layer3 input streams.

- global_controller.v
  Frame-level controller that handles frame_start, frame_idx, and image-level done pulses.

- layer1_stream_ctrl.v, layer2_local_ctrl.v, layer3_local_ctrl.v
  Local controllers that manage valid/ready flow, bottom zero-padding, pipeline drain,
  and final output write timing.

- mac_core.v, q8_8_unit.v, relu.v
  Arithmetic helper modules for fixed-point multiplication, Q8.8 conversion, bias add,
  saturation, and ReLU.


Line Buffer Optimization
------------------------
The initial line buffer idea used row/column address selection over a 3x150 buffer. This
caused large mux structures during synthesis because each 3x3 window element had to select
from many possible row positions.

The final implementation uses a streaming delay-line structure:

- delay1[150] stores the previous row.
- delay2[150] stores the row before the previous row.
- Three 3-element shift registers build the top, middle, and bottom rows of the 3x3 window.

This reduces the large dynamic address-selection muxes and makes the design more suitable
for FPGA implementation.


Padding Strategy
----------------
The SRCNN convolution uses padding=1.

- Top, left, and right padding are inserted inside the line buffer window-generation logic.
- Bottom padding is generated by the layer local controllers by feeding one additional
  zero row after the last real row.


Board Verification Stability Fixes
----------------------------------
Several final changes were applied for board-level stability:

- input_mem_3img.v removes the large full-memory zero-initialization loop and uses direct
  readmemh loading for the three image address ranges.
- Layer3 output write signals are registered inside layer3_local_ctrl.v.
- The final last-pixel signal used for done generation is delayed by 32 cycles so that
  the PS side does not read the output buffer before the final URAM write is safely settled.
- top_srcnn.v packs four signed 16-bit output pixels into one 64-bit word.


Verification Summary
--------------------
Simulation verification compares Layer1, Layer2, Layer3, and final output against the
reference data.

- Layer1 and Layer2 match the reference after aligning the dump order to channel-major.
- Layer3/final output is one channel and matches the row-major reference directly.
- Final output mismatch count: 0.

Implementation summary from the final design document:

- LUT: 43,816
- FF: 76,504
- BRAM: 32
- URAM: 5
- DSP: 396
- WNS: +2.308 ns
- TNS: 0.000
- Failed routes: 0

Processing time estimate:

- About 69,501 cycles for 3 images after the done-delay stabilization.
- About 0.695 ms at 100 MHz.
